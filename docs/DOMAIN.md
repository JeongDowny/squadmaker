# DOMAIN.md

## Purpose
이 문서는 프로젝트 전반에서 사용하는 축구 도메인 개념과 서비스 내부 개념을 고정하기 위한 문서다.

추천 로직, UI 표현, 저장 구조가 서로 어긋나지 않도록 아래 용어를 공통 기준으로 사용한다.

## Core concepts

### Match
경기는 사용자가 생성하는 운영 단위다.

한 경기는 아래 정보를 가진다.
- 경기 제목
- 경기 날짜
- 총 쿼터 수
- 경기 참가 선수 목록
- 쿼터별 포메이션
- 최종 저장된 배치
- 선수별 출전 쿼터 수 요약
- 쿼터별 평균 점수 요약

이 프로젝트는 팀 중심보다 경기 중심 구조를 사용한다.

### Player Pool
선수 풀은 사용자가 보유한 전체 등록 선수 집합이다.

- 선수 풀은 경기와 별개로 유지된다.
- 경기 생성 시 선수 풀에서 참가 선수를 선택한다.
- 추천 계산은 전체 선수 풀이 아니라 경기 참가 선수 기준으로 수행한다.

### Match Participant
경기 참가 선수는 특정 경기에 실제로 포함된 선수다.

- 선수 풀에서 선택된다.
- 각 쿼터의 선발 또는 후보가 될 수 있다.
- 추천 계산 대상은 경기 참가 선수다.

### Quarter
쿼터는 경기 내 시간 블록을 의미한다.

이 서비스에서는 축구 경기 운영 관행에 맞춰 “쿼터”라는 용어를 그대로 사용한다.

쿼터에 대한 원칙:
- 사용자가 총 쿼터 수를 직접 입력한다.
- 모든 쿼터는 같은 길이의 시간 단위로 간주한다.
- 출전 시간 계산은 실제 분 단위가 아니라 출전 쿼터 수 기준으로 처리한다.
- 각 쿼터는 서로 다른 포메이션을 가질 수 있다.

### Starter
선발은 특정 쿼터에 실제로 배치된 11명의 선수를 의미한다.

- 각 쿼터는 선발 11명으로 구성된다.
- 추천 계산은 쿼터별 선발 배치를 생성한다.

### Bench
후보는 해당 경기 참가 선수 중 특정 쿼터에서 선발 제외된 선수다.

원칙:
- 후보는 별도 최적화 대상이 아니다.
- 후보는 추천 결과의 부산물로 결정된다.
- 특정 쿼터에서 선발이 아니면 해당 쿼터의 후보로 본다.

## Formation

### Formation representation
포메이션은 `4-3-3`, `4-2-3-1` 같은 문자열 형식으로 저장한다.

예시:
- 4-4-2
- 4-3-3
- 4-2-3-1
- 3-5-2
- 3-4-3
- 4-1-4-1
- 4-5-1
- 4-3-1-2
- 4-1-2-1-2
- 3-4-1-2

### Formation policy
- MVP에서는 템플릿 포메이션만 지원한다.
- 사용자는 각 쿼터마다 템플릿 중 하나를 선택한다.
- 자유 커스텀 포메이션은 MVP 범위에 포함하지 않는다.

## Positions

### Supported player positions
선수 입력 시 사용할 기본 포지션은 아래와 같다.

- LB
- CB
- RB
- CDM
- CM
- CAM
- LW
- RW
- ST

원칙:
- 모든 선수는 주포지션이 반드시 있어야 한다.
- 부포지션은 최대 2개까지 허용한다.
- 포지션 미지정은 허용하지 않는다.

### Position slots
실제 배치 슬롯은 포메이션에 따라 더 세분화된 고정 이름을 사용한다.

예시:
- GK
- LB
- LCB
- CB
- RCB
- RB
- LWB
- RWB
- CDM
- CM
- CAM
- LM
- RM
- LW
- RW
- LST
- ST
- RST

원칙:
- 추천 계산과 UI 표현은 슬롯 이름 기준으로 동작한다.
- 선수 입력 포지션은 비교적 단순한 기본 포지션 집합을 사용한다.
- 포메이션 슬롯은 기본 포지션보다 더 세밀할 수 있다.

## Position fit

### Base fit scores
선수의 포지션 적합성은 아래 기준으로 점수화한다.

- 주포지션: 1.0
- 부포지션: 0.7
- 인접 계열 포지션: 0.4
- 완전 비적합 포지션: 0.1

### Adjacent families
인접 계열 포지션은 완전히 다른 역할은 아니지만, 일부 적응 가능한 인접 관계를 의미한다.

예시 원칙:
- CB ↔ LCB / RCB
- LB ↔ LWB
- RB ↔ RWB
- CDM ↔ CM
- CM ↔ CAM / CDM
- LW ↔ LM
- RW ↔ RM

주의:
- 인접 계열은 “현실적으로 가끔 가능한 이동” 수준으로만 허용한다.
- 완전 다른 역할의 이동은 완전 비적합으로 본다.

예시:
- ST → RB: 완전 비적합
- LB → CAM: 완전 비적합
- CB → ST: 완전 비적합

## Skill score

### Definition
실력 점수는 감독이 평가한 선수의 단일 종합 점수다.

### Rules
- 범위는 1부터 10까지의 정수다.
- 소수점은 허용하지 않는다.
- 미입력 시 기본값 5를 사용한다.
- MVP에서는 공격, 수비, 체력 등 세부 능력치로 분리하지 않는다.

### Meaning
이 점수는 절대적 객관 수치가 아니라, 경기 운영을 위한 감독의 상대적 판단값이다.

## Recommendation concepts

### Fairness
공평성은 선수별 총 출전 쿼터 수가 특정 선수에게 과도하게 몰리지 않도록 하는 기준이다.

### Quarter strength balance
쿼터 평균 점수 균형은 특정 쿼터에만 실력이 몰리지 않도록 하는 기준이다.

### Position fit
포지션 적합성은 각 선수가 배치된 슬롯과 얼마나 잘 맞는지를 의미한다.

## Recommendation variants

### Recommendation A
출전 쿼터 수 공평성을 가장 우선한다.

### Recommendation B
쿼터 평균 점수 균형을 가장 우선한다.

### Recommendation C
주포지션 일치율을 가장 우선한다.

## Saved result
저장 대상은 추천안 자체가 아니라, 사용자가 수정한 최종 배치다.

저장되는 것은 아래와 같다.
- 경기 정보
- 참가 선수 목록
- 쿼터별 포메이션
- 쿼터별 최종 선발 배치
- 선수별 출전 쿼터 수 요약
- 쿼터별 평균 점수 요약

## Export
이미지 내보내기는 PNG만 지원한다.

원칙:
- 현재 보고 있는 쿼터만 내보내거나
- 전체 쿼터를 별도 요약 화면에서 내보낼 수 있다.
- PNG에는 필드 배치만 포함한다.
