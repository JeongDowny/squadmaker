# PLANS.md

## Purpose
이 문서는 제품의 현재 단계, MVP 범위, 우선순위, 다음 단계 확장 방향을 요약한 로드맵 문서다.

세부 구현 계획은 `docs/exec-plans/active/*` 문서에서 관리하고, 이 문서는 큰 방향과 마일스톤을 정리한다.

## Current product stage
현재 단계는 MVP 설계 및 초기 구현 단계다.

핵심 목표:
- 경기 전 쿼터별 선발 배치 문제를 빠르게 해결하는 제품의 핵심 가치 검증
- 입력 → 추천 → 수정 → 저장의 기본 흐름 완성
- 모바일 사용성을 해치지 않는 첫 버전 완성

## MVP focus
MVP에서 반드시 완성해야 하는 핵심 기능은 아래와 같다.

1. 선수 카드 입력
2. 전체 등록 선수 풀 관리
3. 경기 생성
4. 경기 제목 및 날짜 입력
5. 경기 참가 선수 선택
6. 쿼터 수 입력
7. 쿼터별 포메이션 선택
8. 추천안 A/B/C 생성
9. 쿼터별 선발 11명 배치 보기
10. 선수별 출전 쿼터 수 표시
11. 쿼터별 평균 점수 표시
12. 수동 교체
13. 교체 후 실시간 재계산
14. 비로그인 localStorage 저장
15. 로그인 사용자 경기 저장
16. PNG 내보내기

## Out of scope for MVP
아래 항목은 MVP 범위에서 제외한다.

- 실시간 경기 분석
- 전술 AI 코칭
- 경기 중 교체 시뮬레이션
- 자유 커스텀 포메이션
- 후보 최적화
- 협업 편집
- 공유 링크 기능
- 세부 능력치 기반 추천
- 자동 상대 분석
- 9인 축구 확장

## Current system rules to preserve
MVP 구현 중 아래 원칙은 유지한다.

- 감독이 쿼터별 포메이션을 먼저 선택한다.
- 서비스는 해당 포메이션 안에서 선발 배치를 추천한다.
- 추천 계산은 브라우저에서 수행한다.
- 저장 대상은 추천안이 아니라 최종 수정본이다.
- 비로그인 사용자는 경기 3개까지 저장한다.
- 로그인 사용자는 경기 3개까지 저장한다.
- 이미지 내보내기는 PNG이며 필드 배치만 포함한다.

## Milestones

### Milestone 1: Domain and document foundation
목표:
- 상위 문서 구조 확정
- 도메인 개념 고정
- 추천 규칙 문서화
- AGENTS.md, PRODUCT_SENSE.md, ARCHITECTURE.md, DOMAIN.md 작성

완료 조건:
- 핵심 문서 초안 완성
- 문서 우선순위와 실행 규칙 합의 완료

### Milestone 2: Input and setup flow
목표:
- 선수 풀 입력 및 관리
- 경기 생성
- 경기 제목/날짜/쿼터/포메이션 입력
- 예시 팀 체험 흐름 구성

완료 조건:
- 사용자가 입력 단계를 끝낼 수 있음
- 기본 검증 동작함
- 추천 계산을 호출할 수 있음

### Milestone 3: Recommendation engine MVP
목표:
- 추천안 A/B/C 생성
- 공평성 / 평균 점수 / 적합성 기반 계산 로직 구현
- 기본 지원 포메이션 반영

완료 조건:
- 추천안 3개가 생성됨
- 각 안의 성격 차이가 보임
- 주요 테스트 케이스 통과

### Milestone 4: Quarter lineup editor
목표:
- 쿼터 탭 전환
- 필드 배치 보기
- 수동 교체
- 재계산 반영

완료 조건:
- 사용자가 추천 결과를 직접 수정할 수 있음
- 교체 후 출전 쿼터 수와 평균 점수가 갱신됨
- 모바일 탭 기반 교체 가능

### Milestone 5: Save and export
목표:
- 비로그인 localStorage 저장
- 로그인 DB 저장
- PNG 내보내기

완료 조건:
- 비로그인 사용자 경기 3개 저장 가능
- 로그인 사용자 경기 3개 저장 가능
- 필드 배치 PNG 저장 가능

### Milestone 6: Mobile quality and hardening
목표:
- 모바일 UX 정리
- 오류 메시지 정리
- 저장 실패 fallback 정리
- 문서 정합성 확인

완료 조건:
- 모바일에서 핵심 흐름 사용 가능
- 주요 실패 상황 메시지 확인 완료
- 문서와 구현이 일치함

## Success criteria for MVP release
MVP 출시는 아래를 만족해야 한다.

- 사용자가 선수 입력 후 추천 결과를 빠르게 볼 수 있다.
- 선수별 출전 쿼터 수가 어느 정도 공평하게 분배된다.
- 쿼터별 평균 점수 편차가 지나치게 크지 않다.
- 사용자가 추천 결과를 수동 수정할 수 있다.
- 모바일에서도 핵심 기능을 그대로 사용할 수 있다.

## After MVP
MVP 이후 우선 검토할 확장 항목은 아래와 같다.

1. 9인 축구 확장
2. 더 다양한 포메이션 템플릿
3. 경기 히스토리 비교
4. 시즌별 운영 기능
5. 포지션 세부 능력치
6. 후보 최적화
7. 고급 추천 방식
8. 동호회 운영 툴 기능 확대

## Planning process rule
- 큰 기능 작업은 `docs/exec-plans/active/`에 별도 계획 문서를 만든다.
- 완료된 계획은 `docs/exec-plans/completed/`로 이동한다.
- 모든 비사소한 구현 변경은 execution log에 기록한다.
