# DESIGN.md

## Purpose
이 문서는 제품의 시각적 방향성과 UI 설계 원칙을 정의한다.

이 프로젝트는 축구 서비스이지만, 전체 인상은 거친 스포츠 앱보다 깔끔한 SaaS 제품에 가깝게 가져간다.

## Visual direction
전체 톤은 아래 두 가지를 결합한다.

- 기본 제품 인상: 깔끔한 SaaS
- 핵심 결과 화면: 잔디 느낌을 가미한 전술 보드

원칙:
- 과한 스포츠 감성보다 정보 전달을 우선한다.
- 장식보다 가독성과 구조를 우선한다.
- 전술판 느낌은 결과 화면에서만 강하게 드러나게 한다.

## Primary screen
가장 중요한 화면은 쿼터별 포메이션 결과 화면이다.

이 화면은 사용자가 아래를 한눈에 파악하게 해야 한다.
- 지금 보고 있는 쿼터
- 선발 11명의 배치
- 선수별 출전 쿼터 수
- 평균 점수 상태

## Information hierarchy
결과 화면의 정보 우선순위는 아래와 같다.

1. 필드 배치
2. 선수별 출전 쿼터 수
3. 평균 점수

이 순서는 화면 배치, 강조도, 여백 설계에 모두 반영한다.

## Field board principles
필드는 실제 경기장 묘사보다 전술 보드 느낌을 기준으로 한다.

원칙:
- 선수 슬롯 위치가 명확해야 한다.
- 배치 상태를 빠르게 읽을 수 있어야 한다.
- 잔디 배경은 분위기를 주되 텍스트 가독성을 해치지 않아야 한다.
- 슬롯 간 간격과 좌우 대칭이 안정적으로 보여야 한다.

## Layout principles
### Desktop
- 중앙에 필드 배치를 두고
- 주변에 보조 정보 패널을 배치한다.
- 쿼터 탭은 상단 고정 또는 상단 가까운 위치에 둔다.

### Mobile
- 한 손 사용성을 우선한다.
- 기본은 세로 흐름을 사용한다.
- 필드 배치를 먼저 보여주고, 출전 쿼터 카드와 평균 점수는 그 아래에 둔다.
- 탭 전환과 교체 액션은 손가락으로 쉽게 닿는 위치에 둔다.

## Player presentation
선수는 결과 화면에서 단순 텍스트보다 카드 또는 배지 형태로 표현한다.

원칙:
- 필드 위 선수 표시는 짧고 명확해야 한다.
- 출전 쿼터 수는 카드형 리스트로 별도 표시한다.
- 선택됨, 교체 대상, 출전 부족 상태는 분명히 구분한다.

## Color principles
색상은 상태 표현에 사용하되 과도하게 많아지지 않게 한다.

기본 원칙:
- 주색은 깔끔한 중성 계열 + 포인트 컬러 조합
- 필드 화면은 초록 계열을 쓰되 채도를 낮춰 정보 가독성을 유지
- 위험/부족 상태는 빨간색 계열로 표현

현재 확정 상태:
- 출전 부족은 빨간색으로 표시

추가 원칙:
- 색상만으로 상태를 구분하지 않는다.
- 아이콘, 텍스트, 테두리, 배경 변화와 함께 사용한다.

## Typography principles
- 숫자 정보는 빠르게 비교 가능해야 한다.
- 쿼터 번호, 평균 점수, 출전 횟수는 읽기 쉬운 크기와 굵기로 표시한다.
- 필드 내부 선수 이름은 길이를 제한하거나 축약 규칙을 둔다.
- 작은 화면에서도 선수명과 상태가 겹치지 않아야 한다.

## Interaction design principles
- 사용자는 결과를 보고 곧바로 수정할 수 있어야 한다.
- 수정 흐름은 설명이 없어도 이해 가능해야 한다.
- 두 번 탭하는 교체 방식은 현재 단계와 다음 행동이 명확해야 한다.

예:
- 1차 선택 시 “교체 대상 선택됨” 표시
- 2차 선택 전까지 취소 가능
- 잘못 누른 경우 쉽게 원복 가능

## Recommendation card design
추천안 A/B/C는 성격이 다른 카드로 보여준다.

각 카드에는 최소 아래 정보가 필요하다.
- 추천안 이름
- 핵심 성격
- 선수별 총 출전 쿼터 수 요약
- 쿼터별 평균 점수 요약
- 주포지션 일치율

원칙:
- 숫자가 너무 많아 복잡해지지 않게 한다.
- 어떤 안이 어떤 장점이 있는지 한눈에 보여야 한다.

## Summary view design
전체 쿼터 요약 화면은 기본 결과 화면과 목적이 다르다.

요약 화면 목적:
- 전체 경기의 흐름을 빠르게 점검
- 쿼터 간 차이 비교
- 이미지 내보내기 전 최종 확인

원칙:
- 기본 편집 화면보다 비교가 쉬워야 한다.
- 한 쿼터씩 자세히 보기보다 전체 구조를 파악하게 해야 한다.

## Export design principles
PNG 내보내기는 필드 배치만 포함한다.

원칙:
- 내보낸 이미지가 깔끔하고 바로 공유 가능해야 한다.
- 불필요한 버튼, 폼, 편집 UI는 포함하지 않는다.
- 제목과 쿼터 정보가 필요하면 최소 정보만 넣는다.
- 출력 이미지 비율은 모바일 캡처 느낌보다 공유용 카드 느낌에 가깝게 한다.

## Design system guidance
- 공통 카드, 버튼, 탭, 배지, 상태 표시 패턴을 먼저 정의한다.
- 기능마다 독자적 스타일을 새로 만들지 않는다.
- 동일한 상태는 항상 동일한 디자인 신호를 쓴다.

## Non-goals
이 문서는 아래를 목표로 하지 않는다.
- 실제 방송 그래픽 수준의 경기장 재현
- 화려한 애니메이션 중심 UI
- 게이밍 UI 스타일
