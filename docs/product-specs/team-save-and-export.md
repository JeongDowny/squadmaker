# team-save-and-export.md

## Goal
사용자가 수정 완료한 경기 배치를 안전하게 저장하고, 필요 시 공유용 PNG 이미지로 내보낼 수 있도록 한다.

이 문서는 저장과 내보내기 기능의 범위와 동작 원칙을 정의한다.

## Scope
포함 범위:
- 비로그인 저장
- 로그인 저장
- 저장 가능 개수 제한
- 최종 수정본 저장 정책
- PNG 내보내기

포함하지 않는 범위:
- 공유 링크
- 협업 편집
- PDF export
- 추천안 자체의 다중 저장
- 공개 갤러리

## Save model
저장 대상은 추천안 원본이 아니라, 사용자가 최종적으로 수정한 경기 배치다.

저장되는 핵심 정보:
- 경기 제목
- 경기 날짜
- 참가 선수 목록
- 총 쿼터 수
- 쿼터별 포메이션
- 쿼터별 최종 선발 배치
- 선수별 출전 쿼터 수 요약
- 쿼터별 평균 점수 요약

## Logged-out save
비로그인 사용자는 localStorage 기반으로 저장한다.

정책:
- 저장 가능한 경기는 1개
- 저장은 임시 저장 성격으로 본다
- 브라우저 환경에 종속된다
- 다른 기기와 동기화되지 않는다

원칙:
- 저장 제한을 사용자에게 명확히 보여준다
- 기존 저장 경기 덮어쓰기 시 사용자에게 알린다
- localStorage 저장 실패 시 안내 메시지를 보여준다

## Logged-in save
로그인 사용자는 DB 기반으로 저장한다.

정책:
- 구글 로그인만 지원
- 저장 가능한 경기는 최대 3개
- 경기 3개 제한은 서버에서도 검증해야 한다
- 사용자는 자신의 경기만 저장, 수정, 조회할 수 있어야 한다

## Save action triggers
저장은 아래 상황에서 발생할 수 있다.

- 사용자가 수동으로 저장 버튼을 누름
- 저장 실패 후 재시도
- 로그인 전 임시 작업을 로그인 후 저장으로 전환

자동 저장은 MVP 필수 범위에 포함하지 않는다.

## Save overwrite rules
저장 동작은 아래 두 경우를 구분한다.

### Create new match save
- 아직 저장되지 않은 경기를 새로 저장하는 경우

### Update existing match save
- 이미 저장된 경기를 다시 수정해 저장하는 경우

원칙:
- 생성과 수정은 UI에서 구분 가능해야 한다
- 저장 가능 개수 제한은 새 저장 시점에만 적용한다
- 기존 경기 수정은 저장 개수 제한과 별개로 허용한다

## Save failure handling
저장 실패 시 아래 원칙을 따른다.

1. 현재 편집 상태를 잃지 않는다.
2. 가능한 경우 브라우저 임시 저장을 시도한다.
3. 사용자에게 실패 사실과 다음 행동을 알려준다.

기본 메시지 예시:
- 저장 실패, 브라우저에 임시 저장됨.

## Export model
내보내기는 PNG만 지원한다.

원칙:
- 사용자는 현재 보고 있는 쿼터 기준으로 PNG를 내보낼 수 있다.
- 전체 쿼터 요약 화면에서도 PNG 내보내기를 지원할 수 있다.
- 내보내기 이미지에는 필드 배치만 포함한다.

## Export content rules
PNG에 포함되는 것:
- 쿼터 필드 배치
- 필요한 경우 최소한의 쿼터 식별 정보

PNG에 포함되지 않는 것:
- 버튼
- 입력 폼
- 편집 UI
- 선수별 출전 쿼터 카드 리스트
- 평균 점수 상세 정보
- 저장 액션 UI

## Export UX requirements
- 내보내기 결과는 바로 공유 가능한 수준으로 깔끔해야 한다.
- 결과물은 모바일 캡처가 아니라 정돈된 카드형 이미지에 가까워야 한다.
- 글자 잘림이나 레이아웃 깨짐이 없어야 한다.

## Edge cases
### Logged-out user tries to save second match
- 기존 localStorage 저장 경기 존재 시 안내 필요
- 덮어쓰기 또는 현재 경기 유지 정책을 명확히 보여줘야 한다

### Logged-in user exceeds 3 saved matches
- 새 경기 저장을 막고 안내 메시지를 보여준다
- 기존 경기 수정 저장은 허용한다

### Export failure
- 렌더링 실패 또는 이미지 생성 실패 시 재시도 안내를 제공한다
- 편집 중인 화면 상태는 유지한다

## Non-goals
이 문서는 아래를 다루지 않는다.
- 추천 계산 로직
- 경기 중 수정 흐름
- 협업 저장 모델
