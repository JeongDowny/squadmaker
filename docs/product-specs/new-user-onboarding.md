# new-user-onboarding.md

## Goal
처음 방문한 사용자가 이 서비스가 무엇을 하는지 빠르게 이해하고, 짧은 단계 안에 첫 결과를 보도록 돕는다.

## Target user
이 온보딩은 주로 아래 사용자를 위한 것이다.

- 축구 동호회 회장
- 경기 운영을 맡는 감독
- 포메이션과 역할 구분은 가능하지만, 여러 쿼터 배치를 수동으로 짜는 데 시간을 많이 쓰는 사용자

## Main onboarding promise
이 서비스는 선수별 출전 쿼터 수와 쿼터별 실력 균형을 고려한 선발 배치를 빠르게 만들어준다는 점을 첫 경험에서 명확히 보여줘야 한다.

## Entry points
첫 진입 시 사용자가 선택할 수 있는 대표 진입점은 아래 두 가지다.

1. 예시 팀으로 체험하기
2. 내 경기 만들기

## Example team experience
예시 팀 체험의 목적은 아래 세 가지다.

- 서비스 이해
- UI 체험
- 결과물 맛보기

예시 팀 체험을 통해 사용자는 아래를 빠르게 확인해야 한다.
- 쿼터별 포메이션이 어떻게 보이는지
- 추천안 A/B/C가 어떻게 다른지
- 선수별 출전 쿼터 수가 어떻게 표시되는지

## Primary onboarding flow
실제 사용 흐름은 2단계로 설계한다.

### Step 1
선수 입력 + 경기 설정
- 경기 제목
- 경기 날짜
- 참가 선수 입력 또는 선택
- 선수 이름
- 주포지션
- 부포지션 최대 2개
- 실력 점수
- 총 쿼터 수
- 쿼터별 포메이션 선택

### Step 2
추천 결과 확인
- 추천안 A/B/C 확인
- 쿼터별 배치 확인
- 선수별 출전 쿼터 수 확인
- 평균 점수 확인
- 수정 시작

## Minimum input for first result
첫 추천 결과를 보기 위한 최소 입력은 아래와 같다.

- 경기 제목
- 경기 날짜
- 참가 선수
- 각 선수의 이름
- 각 선수의 주포지션
- 각 선수의 부포지션 최대 2개
- 총 쿼터 수
- 쿼터별 포메이션

실력 점수는 입력하지 않아도 되지만, 이 경우 기본값 5가 적용된다.

## Login guidance
이 서비스는 비로그인 상태에서도 사용할 수 있다.

하지만 아래 시점에서 로그인 유도를 한다.
- 예시 팀 체험 후 내 경기로 넘어갈 때
- 저장 기능을 사용하려 할 때
- 여러 경기 관리가 필요할 때

로그인 방식은 구글 로그인만 지원한다.

## Logged-out experience
비로그인 사용자는 아래를 할 수 있다.
- 예시 팀 체험
- 경기 입력
- 추천 결과 확인
- 수정
- localStorage에 경기 1개 저장

제한:
- 장기 저장 보장 없음
- 저장 가능한 경기 수 1개

## Logged-in experience
로그인 사용자는 아래를 할 수 있다.
- 경기 3개 저장
- 여러 경기 관리
- 저장된 경기 재진입

## UX principles
- 첫 화면에서 서비스 목적이 바로 이해되어야 한다.
- 입력 전에 복잡한 설명을 길게 보여주지 않는다.
- 예시 팀은 설명보다 체험 중심으로 구성한다.
- 첫 결과는 최대한 빨리 보여줘야 한다.
- 로그인은 막는 단계가 아니라 저장을 위한 가치 제안으로 보여줘야 한다.

## Success criteria
온보딩은 아래를 만족해야 한다.

- 사용자가 서비스 목적을 빠르게 이해한다.
- 첫 결과까지 흐름이 짧다.
- 예시 체험 후 실제 사용으로 자연스럽게 이어진다.
- 로그인 필요성이 강요가 아니라 이점으로 이해된다.

## Non-goals
이 문서 범위에는 아래가 포함되지 않는다.
- 상세 튜토리얼
- 경기 중 운영 안내
- 고급 전술 설명
